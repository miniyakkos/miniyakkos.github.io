Week 5 Technical Blog

Ruby classes

1/25/15

This week I learned a lot about classes. I've watched CS lectures online so I had a general idea of what they were. Classes are a way of creating multiple objects that can respond to the same sets of methods defined inside a class. This might be a difficult concept to understand at first, so let's use an example. Let's pretend we're making an awesome video game with different classes of people. Let's make a class called mage!

First thing first, we'll need to create the class:

class Mage
end

This class does nothing at the moment. We can define an initialize method that gives certain characteristics to all Mages that are created. Let's say in our game, all Mages start at level 1.

class Mage
def initialize
@level = 1
end
end

The @ marks a variable as an instance variable. This means that it is a variable that belongs to a single instance. In this case, one Mage's level doesn't interfere with another Mage's level, since all Mages are different instances. It also means that any method inside the class Mage can access that variable.

So what exactly makes a Mage a Mage? They are able to cast a magic spell! Let's add that ability to the class.

class Mage
def initialize
@level = 1
end

def magic_spell
casts fireball
end
end

The method magic_spell can be used only by people of class Mage. Let's create a new Mage.

Mage.new

And that's how you create a new Mage.

Now that we have a class called Mage, we can keep defining methods within the class to make Mages unique. We can have advanced spells that are accessible to only experienced Mages who are level 20 or greater. We can train our Mages to gain levels by fighting monsters. In that case, we can create a new class called Monster. The possibilities are endless.
