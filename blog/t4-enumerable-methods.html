Enumerable Methods

1/18/15

Enumerable Methods are very useful when it comes to sorting and searching through a collection. One of the methods is #map. The map method runs a block of code following the method and returns the results in an array. (If you're unfamiliar with arrays, take a look at my previous blog post here). Here's an example of the map method in action:

[1, 2, 3, 4, 5].map { |x| x + 10 }

This method returns an array of numbers 11 to 15, or [11, 12, 13, 14, 15]. It adds 10 to each element of the array. The x is just a placeholder so the method can run the block of code (x + 10) for each element, x. The original array is unmodified, the map method returns a new array. This means that if we assign the original array to a variable and run map like this:

arr = [1, 2, 3, 4, 5]
arr.map { |x| x + 10 }

The original array remains as [1, 2, 3, 4, 5]. If you want to replace the original array, using #map! would do the trick.
